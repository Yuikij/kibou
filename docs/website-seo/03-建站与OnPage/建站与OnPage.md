## 域名选择与网站结构：奠定SEO基石

在开始任何开发工作前，正确的域名选择和网站结构设计是决定SEO长期成功与否的关键地基。错误的起点会导致后续大量返工。

### 1.1 域名选择的核心策略
- **域名年龄的真相**：谷歌官方已明确表示，域名年龄本身不是排名因素。若考虑购买老域名，**必须**使用Whois历史查询工具（如 `whois.domaintools.com`）仔细核查其过往记录，避免继承被惩罚的“黑历史”。
- **域名关键词**：包含关键词的作用已大幅减弱。优先选择**简单易记、与品牌强相关、能清晰传达业务**的域名。
- **域名后缀**：面向全球市场，`.com` 是首选。针对特定国家市场（如英国、德国），使用对应的国家代码顶级域名（如 `.uk`、`.de`）能传递更强的地理信号。
- **注册期限**：注册期限过短可能被谷歌视为低信任度信号。建议**一次性注册2-5年**，以表明网站长期运营的意图。
- **主域名设定与重定向**：
    1.  **确定主域名形式**：强烈建议选择不带 `www` 的版本（如 `example.com`）作为主域名。用户习惯直接输入此形式，且能确保所有反向链接权重直接累积于此。
    2.  **必须设置301重定向**：将带 `www` 的版本（`www.example.com`）通过301重定向跳转到主域名，合并权重，避免分散。

### 1.2 网站结构与多语言规划
清晰的结构帮助谷歌高效抓取和理解内容，同时提升用户体验。

- **多语言网站的结构选择**：
    - **核心结论：强烈建议使用子目录（如 `example.com/zh/`）而非子域名（如 `zh.example.com`）。**
    - **原因**：谷歌将子域名视为独立网站，需从零积累权重。使用子目录则能共享主域名权重，新语言页面能继承主站信任，冷启动更快。
    - **技术实现**：每种语言必须有独立的URL，不能仅靠前端JS切换。需为每种语言页面设置正确的 `hreflang` 和 `canonical` 标签。
- **网站层级与导航**：
    - **主导航设计**：应基于**核心业务关键词**和**用户高频需求**来规划菜单项。上线后，需根据Google Search Console (GSC) 和 Google Analytics (GA) 数据，将高流量页面纳入主导航。
    - **面包屑导航**：是有效的“位置指示器”（如“首页 > 分类 > 当前页”），能降低跳出率，传递页面层级权重。
    - **URL设计原则**：简短、清晰、包含目标关键词，使用连字符分隔单词。避免堆砌关键词或使用奇怪字符。
- **分页与页脚**：
    - 分类页需分页时，确保每个分页有唯一URL（如 `?page=2`），并用 `<a>` 标签链接。对筛选、排序产生的重复URL使用 `noindex`。
    - **页脚必须包含**：关于我们、联系方式、隐私政策、服务条款等信任页面，以及订单跟踪、站点地图等用户体验页面。

### 1.3 技术部署基础
- **HTTPS是强制项**：直接影响安全信任和排名，必须启用。
- **移动端友好**：超过一半的流量来自移动端。标准适配代码为 `<meta name="viewport" content="width=device-width, initial-scale=1"/>`。
- **推荐部署方案**：使用 **Cloudflare** 进行DNS解析和安全加速，配合 **Vercel** 或 **Cloudflare Pages** 部署静态网站，免费套餐足以满足初期需求。
- **页面速度优化**：图片是加载慢的主因。上传前必须压缩，产品缩略图尤其要优化。视频建议使用YouTube等平台托管并嵌入，可设置延迟加载。
- **必备文件与图标**：
    - 使用 [favicon.io](https://favicon.io/favicon-converter/) 一键生成多尺寸网站图标（Favicon）和 `site.webmanifest` 文件，并将代码加入 `<head>`。
    - 正确配置 `robots.txt`，避免错误屏蔽重要页面。

## 页面级OnPage优化：打造高排名内容

这是SEO的核心战场，决定了单个页面能否精准匹配搜索意图并赢得排名。

### 2.1 TDK与内容结构：页面的信号骨架
谷歌已不使用 `<meta name="keywords">`，优化重点是 **TDH（Title, Description, Headings）**。

- **Title（标题标签）**：
    - **权重极高**，是仅次于页面内容的第二大排名因素。
    - **规则**：每页**独特**；长度 **50-60字符**；**重要关键词前置**；可在末尾加入品牌名。
    - **格式示例（内页）**：`核心功能-子栏目-网站名`
- **Description（元描述）**：
    - 非直接排名因素，但极大影响点击率。
    - **规则**：每页**独特**；长度约 **150-160字符**；自然包含关键词，可加入号召性用语。
    - **避免**：为所有产品页使用完全相同的描述模板，这会导致内容重复。
- **Headings（标题标签 H1-H6）**：
    - **每个页面有且只有一个H1**，用于概括核心主题。H2-H6用于划分内容板块，形成清晰的逻辑结构。
- **提升点击率的技巧**：在标题和描述中可加入数字（如“15种…”）、疑问句、情感词汇或明确利益点（如“免费”、“指南”）。

### 2.2 内容质量与内链策略：构建价值网络
- **内容为王**：
    - **原创与深度**：坚决抵制搬运抄袭。AI可用于生成初稿，但**必须**人工修改、补充真实数据和见解。内容需全面、实用，能彻底解决用户搜索该关键词时的所有疑问。
    - **关键词密度**：自然融入，无绝对标准。可参考竞品，但**严禁堆砌**。使用 **AITDK浏览器插件** 等工具检查关键词密度是否合理。
    - **内容长度门槛**：一个SEO友好的页面，英文单词数量**不应低于600个**，**建议800个以上**。当新页面曝光低时，应优先检查内容长度。
- **内链建设**：
    - **重要性不亚于外链**。内链是网站的脉络，传递权重、加速收录、提升用户停留。
    - **核心策略**：构建清晰的树形结构（首页→分类→内页），确保任何页面在 **3-4次点击** 内可达。**在首页或栏目页列出最新/最重要页面**是加速新页面收录的有效方法。
    - **最佳实践**：锚文本多样化；新页面既链接旧页面，也要在相关旧页面中添加指向新页面的链接；优先使用文本链接。
- **出站链接**：链接到权威相关网站能丰富页面主题、增强权威性，但需合理使用，可为不传递权重的链接设置 `nofollow`。

### 2.3 其他关键页面元素
- **图片优化**：
    - 文件名根据内容描述性命名（如 `blue-running-shoes.jpg`）。
    - **必须添加准确的Alt属性**，这是图片搜索排名和可访问性的关键。
- **结构化数据（Schema Markup）**：通过JSON-LD格式主动提供数据，可在搜索结果中展示评分、FAQ、面包屑等富媒体样式，显著提升点击率。
- **Canonical标签**：用于解决内容重复问题。每个页面的 `canonical` 标签应指向自身唯一的规范URL。**错误地将所有页面指向首页会严重损害SEO**。
- **社交元标签（Open Graph）**：控制页面在社交媒体（如Facebook、Twitter）分享时展示的标题、描述和图片，影响社交流量。

### 2.4 用户体验与行为信号
谷歌通过用户行为（点击率、停留时间、跳出率）验证页面是否满足搜索意图。
- **核心策略**：对于工具类网站，**提供无需登录的免费试用**，能极大降低使用门槛，延长页面停留时间，向谷歌发出强烈的积极用户信号。
- **其他信号**：加载速度、移动端友好、无侵入式插页广告、HTTPS安全等，都是重要的排名因素。

## 高级策略与自动化：规模化获取流量

当基础扎实后，可采用以下策略进行规模化流量获取。

### 3.1 程序化SEO
- **核心思路**：挖掘一个微小但普遍存在的结构化需求（如“两地距离”、“电话号码格式”），通过程序化生成海量页面，每个页面精准满足一个具体的搜索查询。
- **关键四要素**：
    1.  **结构化数据源**：将需求数据整理成可查询的格式。
    2.  **SEO友好模板**：设计一个用于动态填充数据的页面模板。
    3.  **海量页面生成**：通过组合（如200个国家两两配对）生成数万至数百万页面。
    4.  **内链网络构建**：必须建立从首页到各级页面的清晰内链结构，确保爬虫能抓取到所有页面。
- **上量策略**：新站初期每天只生成少量页面（如10个）测试，根据GSC数据反馈调整模板和策略，确认有效后再逐步增加到每天100、1000个。

### 3.2 利用AI原生CMS与编程助手提升效率
- **AI原生CMS理念**：从设计之初就基于AI能力重构网站，实现内容发现、生成、翻译、内链建设的自动化。
- **实践（以AI导航站为例）**：
    - **架构设计**：为每个实体（如AI工具）创建结构化的“页面”，页面由多个可独立优化的“章节”（简介、功能、价格、FAQ）组成。
    - **内容生成**：通过爬虫自动收集产品信息作为“参考资料”，然后用AI（如GPT-4o）基于这些资料和专门的提示词为每个章节生成内容，避免幻觉。
- **使用AI编程助手**：利用 **Windsurf** 等工具，通过分步提示词快速生成包含多模块（Hero区、特色展示、FAQ、页脚等）和交互功能的落地页。使用 `Ctrl+L` 选中代码片段作为上下文，可提高代码生成稳定性。

### 3.3 创建高效SaaS产品落地页
针对SaaS产品，应优先从漏斗底部（BOFU）关键词创建内容，转化率更高。

- **五种核心落地页类型**：
    1.  **变体词落地页**：为核心关键词创建所有同义/近义词的独立页面。
    2.  **竞品对比页**：为“Your Product vs. Competitor”或“Competitor Alternatives”创建独立页面。
    3.  **功能介绍页**：为产品的每个核心功能点创建独立页面。
    4.  **相关小工具页**：将产品功能拆分为可在线免费体验的独立小工具，吸引流量。
    5.  **模板页**：为有模板需求的关键词创建独立页面。
- **落地页关键模块**：必须包含清晰的Hero区（产品描述+CTA）、详细优势、社会证明（至少三种类型，如Logo墙、用户评价、数据案例）、FAQ以及多个保持一致的CTA按钮，可辅以降低决策门槛的文案（如“No credit card required”）。

## SEO工作流与长期架构

### 4.1 标准建站与优化流程
1.  **需求与关键词研究**：使用Semrush、Ahrefs等工具，挖掘低竞争（KD<30）、有搜索量且符合商业意图的关键词。
2.  **规划与建站**：基于关键词规划网站结构。注册域名，配置Cloudflare + Vercel技术栈。使用WordPress + Rank Math等高效工具。
3.  **OnPage优化**：严格遵循TDH规范。创建高质量、原创、满足搜索意图的内容（词数≥800）。系统化建设内链。优化所有图片和多媒体元素。
4.  **上线与收录**：在GSC提交站点地图。在爬虫活跃平台（如V2EX、Reddit）发布链接，吸引爬虫抓取。
5.  **数据驱动迭代**：分析GSC数据，持续优化排名前20的页面。根据数据扩展内容。逐步开展外链建设。考虑多语言或程序化SEO扩展。

### 4.2 长期流量与风险管理
- **流量来源多元化**：
    - 避免将90%的流量集中于首页，应让流量分散到网站众多页面。
    - 同时布局工具页、列表页、博客文章页等不同类型。
    - 在SEO之外，可探索网红营销、广告投放等多渠道。
- **架构管理**：对于拥有海量页面的网站，需分析竞品目录结构，设计清晰的分类目录，并强化重要页面的内链布局。
- **出海实操心态**：
    - **从长尾词入手**：“宁做鸡头不做凤尾”，快速积累成功经验和收入。
    - **观察学习**：即使不自己建站，也要定期观察新域名的注册、上线、排名及外链情况。
    - **提升开发效率**：强烈建议购买现有代码模板进行修改，而非从零开发，缩短MVP上线周期。
    - **快速验证**：海外用户容忍度高，核心功能可运行即可先上线，避免过度开发。
    - **支付风险防控**：必须上线并维护多个支付渠道（如PayPal），避免单一渠道被封导致收入中断。

## 速查清单
- [ ] **域名与部署**：域名已选定并注册（建议不带www），已设置301重定向；HTTPS已启用；已通过Cloudflare解析并部署至Vercel等平台。
- [ ] **网站结构**：多语言方案确定为子目录结构；主导航基于关键词和用户需求设计；面包屑导航已配置；URL结构清晰且包含关键词；页脚包含信任页面链接。
- [ ] **TDH与元数据**：每个页面都有独特、包含核心关键词、长度适中的Title和Description；页面有且仅有一个H1，标题层级（H2-H6）逻辑清晰。
- [ ] **内容与内链**：核心页面英文单词数≥600，建议≥800；内容原创、全面；关键词布局自然，密度合理（使用AITDK检查）；网站内部链接网络清晰，任何页面3-4次点击可达。
- [ ] **页面元素**：所有图片文件名描述性，已添加准确的Alt文本；图片已压缩；设置了正确的Canonical标签；考虑为重要页面添加结构化数据（Schema）。
- [ ] **用户体验**：页面移动设备友好；加载速度已优化；工具类网站提供无需登录的免费试用。
- [ ] **技术验证**：已在Google Search Console验证网站并提交站点地图；已安装Google Analytics；已检查并正确配置robots.txt；已生成并部署Favicon。
- [ ] **高级策略**：若需多语言，已规划使用子目录并设置hreflang标签；已评估是否可采用程序化SEO；已为SaaS产品创建五类关键落地页。
- [ ] **长期规划**：流量来源已规划多元化；支付渠道已设置备份；遵循数据驱动迭代流程。
